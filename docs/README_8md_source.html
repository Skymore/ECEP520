1 Elma is an event loop and process manager for embedded systems. It keeps track of processes, executes them at specified frequencies, and enables interprocess communication.

6 As an example, we will develop a simple cruise control system in which we model a car with one process and a control system with another. The underlying mathematics look like this. Let `v` be the velocity of a car, `m` its mass, `k` the coefficient of rolling friction, and `u` the force being applied by the engine on the car. Then a very simplified model of the car's velocity comes from Newton's second law (`f = ma`):
7 
8  m dv/dt = -k v + u
9 
10 This is a continuous model, but our processes are discrete. We can discretize the model using the definition of derivative:
11 
12  dv/dt = lim(h->0) (v(t) - v(t-h)) / h
13 
14 For `h` small, `dv/dt` is thus approximately
15 
16  dv/dt = (v(t) - v(t-h)) / h
17 
18 or
19 
20  v(t) = v(t-h) + h dv/dt
21  = v(t-h) - h (k v - u)
22 
23 In our case, `h` is given by the difference between the last update and the previous update, which is available to user processes via the `delta()` function. Thus, we can write
24 
25  class Car : public elma::Process {
26  public:
27  Car(std::string name) : Process(name) {}
28  void init() {}
29  void start() {
30  velocity = 0;
31  force = 0;
32  }
33  void update() {
34  if ( channel("Throttle").nonempty() ) {
35  force = channel("Throttle").latest();
36  }
37  velocity += ( delta() / 1000 ) * ( - k * velocity + force ) / m;
38  velocity += ( delta() / 1000 ) * ( - k * velocity + force ) / m;
39  channel("Velocity").send(velocity);
40  std::cout << "t: " << milli_time() << " ms\t"
41  << " u: " << force << " N\t"
42  << " v: " << velocity << " m/s\n";
43  }
44  void stop() {}
45  private:
46  double velocity;
47  double force;
48  const double k = 0.02;
49  const double m = 1000;
50  };

52 A very simple controller uses proportional feedback. Basically, we set
53 
54  u = - Kp ( v - vdes )
55 
56 where `Kp` is the proportional gain and `vdes` is the desired velocity. To make a control process, we write
57 
58  class CruiseControl : public elma::Process {
59  public:
60  CruiseControl(std::string name) : Process(name) {}
61  void init() {}
62  void start() {
63  speed = 0;
64  }
65  void update() {
66  if ( channel("Velocity").nonempty() ) {
67  speed = channel("Velocity").latest();
68  }
69  channel("Throttle").send(-KP*(speed - desired_speed));
70  }
71  void stop() {}
72  private:
73  double speed;
74  const double desired_speed = 50.0,
75  KP = 250.0;
76  };

78 To set everything up, we add the two processes and the two channels to the manager, initialize everything, and run the system.

80  elma::Manager m;
81 
82  Car car("Car");
83  CruiseControl cc("Control");

85  elma::Channel velocity("Velocity");
86 
87  m.schedule(car, MS(10))
88  .schedule(cc, MS(10))
89  .add_channel(throttle)
90  .add_channel(velocity)
91  .init()
92  .run(MS(10000));
93 
94 Note that the order of the scheduling is important. We want the controller to run right after the car. If we reversed the order, the controller would be using an older sensor value.
